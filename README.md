# HappyTrigger

<p align="center">
  <img src="docs/images/icon.png" width="160" alt="HappyTrigger icon">
</p>

HappyTrigger は、FFXIV / Dalamud 向けの **ログ連動ポップアップ表示プラグイン**です。  
チャットログ、バトルログ、内部ログ、VFXログなどを条件にして、画像・テキスト・残り時間つきテキストを画面上へ表示できます。

主な用途は、ギミック処理のコール表示、デバフ残り時間の一覧表示、ログ条件を組み合わせたカスタム通知です。

---

## 画面イメージ

残り時間つきログトリガーを、トリガーラベルの基準座標から縦積み表示できます。

![label stack example](https://raw.githubusercontent.com/Rowsai/HappyTrigger/refs/heads/main/docs/image/label_stack_example.png)

---

## 主な機能

### 1. トリガー表示

- チャットログのキーワード一致で画像を表示
- テキストトリガーで任意文字列を表示
- FFXIVログ参照トリガーで、バトルログ / 内部ログ / VFXログを条件に表示
- 複数ログ条件の組み合わせ判定
- 前提条件トリガー対応
- 表示待機秒数、表示秒数、フェード、座標、サイズを設定可能
- ローカル画像 / Web画像URLに対応

### 2. トリガー管理階層

トリガーは以下の3階層で管理します。

![trigger hierarchy](docs/images/trigger_hierarchy.png)

```text
トリガーボックス（BOX***）
  └ トリガーラベル（Lab***）
      └ ログトリガー（F***** / X***** など）
```

各ログトリガーには任意の名称を設定できます。  
トリガー管理タブでは、ラベル配下のログトリガーは名称を中心に表示します。

### 3. 削除確認

トリガーボックス、トリガーラベル、ログトリガーの削除時には確認ダイアログを表示します。

```text
ID：****　トリガーボックス or トリガーラベル or ログトリガー名：***** を削除します。
よろしいですか？

YES / NO
```

削除範囲は以下の通りです。

| 対象 | 削除されるもの |
|---|---|
| トリガーボックス | ボックス配下のラベル、ラベル配下のログトリガー |
| トリガーラベル | ラベル配下のログトリガー |
| ログトリガー | 対象ログトリガーのみ |

## トリガーラベル座標機能

### 基本仕様

トリガーラベルには、ラベル全体の基準座標を設定できます。

```text
トリガーラベルA
  PositionX = 1000
  PositionY = 1000
  LineSpacing = 任意
```

ラベル座標を使用するログトリガーは、個別の `PositionX / PositionY` ではなく、ラベルの基準座標から縦方向に並びます。

```text
--- トリガーラベルA ---
ログトリガーA
ログトリガーB
ログトリガーC
ログトリガーD
```

このうち A と D だけ条件マッチした場合、未マッチの B / C は空行にせず、表示対象だけを詰めて表示します。

```text
X=ラベルAのX / Y=ラベルAのY          ログトリガーA
X=ラベルAのX / Y=ラベルAのY+1行分    ログトリガーD
```

### ログトリガー側の切り替え

各ログトリガー側で、ラベル座標を使用するかどうかを選択できます。

| 設定 | 動作 |
|---|---|
| ON | トリガーラベルの基準座標から縦積み表示 |
| OFF | ログトリガー自身の個別座標で単独表示 |

同じラベル配下でも混在できます。

```text
ログトリガーA：ON  → ラベル座標から縦並び
ログトリガーB：OFF → 個別座標で単独表示
ログトリガーC：ON  → ラベル座標から縦並び
ログトリガーD：OFF → 個別座標で単独表示
```

## ラベル単位のテスト表示 / 座標設定

トリガー管理タブの階層表示で、トリガーラベル行に以下の操作があります。

```text
テスト
座標設定
削除
```

### テスト

ラベル配下のログトリガーをまとめてテスト表示します。

### 座標設定

ラベル配下のログトリガーをまとめて座標設定表示します。

- ラベル座標ONのログトリガー  
  → ラベル基準座標の設定対象になります。どれか1つをドラッグすると、ラベル全体の基準座標が移動します。

- ラベル座標OFFのログトリガー  
  → そのログトリガーだけの個別座標設定対象になります。

右クリックで座標設定表示を閉じます。

## 残り時間表示と並び順

ログトリガーにステータス残り時間表示を設定できます。

設定項目例：

```text
ステータスの残り時間を取得する
残り時間参照ジョブ
残り時間参照ステータス名
他人に付与されたバフを同時表示する
```

トリガーラベル座標ONのログトリガーで、残り時間表示がある場合は、**残り時間が短いものほど上に優先表示**します。

表示優先度は以下です。

```text
1. 残り時間表示あり
   → 残り時間が短い順

2. 残り時間表示なし
   → 従来どおりID順
```

例：

```text
ログトリガーA：54.10s
ログトリガーB：31.20s
ログトリガーC：40.40s
ログトリガーD：15.30s
```

表示順：

```text
ログトリガーD：15.30s
ログトリガーB：31.20s
ログトリガーC：40.40s
ログトリガーA：54.10s
```

残り時間表示なしのログトリガーは、残り時間ありの表示群の後ろにID順で表示されます。

## FFXIVログ参照トリガー

ログトリガーでは、バトルログと内部ログの条件を組み合わせて判定できます。

例：

```text
バトルログ条件：敵の詠唱ログ
内部ログ条件1：MemberStatus Information
内部ログ条件2：StatusName=水属性圧縮
```

複数条件は指定秒数内に揃った場合のみ発火します。  
内部ログ条件はゲーム側の出力順が前後しても判定できるように、順番不問で扱います。

---

## VFXログ

VFXログ収集機能により、Actor VFX の `.avfx` パスを内部ログへ出力できます。  
Splatoon系のレイアウトで使用される VFX パスを、HappyTrigger のログ条件として利用できます。

---

## VOICEVOX読み上げ

テキスト表示トリガーでは、表示テキストを VOICEVOX Engine で読み上げできます。

- VOICEVOX Engine が起動していない場合でも、トリガー表示は継続します
- 音声再生に失敗した場合は、内部ログにエラーを出力します
- Endpoint / Speaker ID を設定可能です

## コマンド

```text
/happytrigger
```
